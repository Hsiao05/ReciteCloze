# 背诵·完形（ReciteCloze）

这是一个用来背诵文本和闪卡的小工具。它的主要思路是：先把需要背的内容切成小块，然后把重要内容遮起来，使用者可以一层一层地背，也可以只练重点内容。

这个项目目前是一个纯前端网页工具，打开网页后就能用，卡片、分组、学习记录会保存在当前浏览器里。如果换浏览器、换电脑，或者清理浏览器数据，需要提前导出文件备份。

## 主要功能

- 文本背诵：粘贴一段文本后，可以生成带遮盖层级的标注内容。
- 隔字生成：没有 API Key 的时候，也可以用本地的简单规则生成遮盖内容。
- DeepSeek 生成标注：可以填写自己的 DeepSeek API Key，让模型帮助判断哪些内容更适合背。
- 标注结果可编辑：生成后的标注可以手动修改，再重新解析进入背诵。
- 多层遮盖：支持原文、一层、二层、重点、全遮。
- 手动重点：背诵时可以长按词块，把某个内容标为重点。
- 闪卡背诵：可以从 CSV 导入，也可以手动录入字段和卡片。
- 类 Anki 学习流程：闪卡支持正面、反面、总览、重点，以及重来、困难、良好、简单。
- 卡片缓存：生成过的背诵内容会自动保存到浏览器里。
- 文件夹管理：可以把不同卡片放进不同文件夹。
- 导入导出：可以把卡片导出成 JSON，也可以从本地 JSON 或公开 GitHub 链接导入。
- 深色模式：右上角可以切换浅色和深色显示。
- 字号调节：背诵页面可以调整文字大小。

## 适合用来做什么

这个工具比较适合下面这些场景：

- 背历史、政治、地理、生物、概念解释这一类需要记关键词的内容。
- 背英语、日语、中文材料里的句子和术语。
- 把课堂笔记变成遮盖背诵材料。
- 把单词表、问答表、概念表做成闪卡。
- 在手机或电脑浏览器里快速复习，不想安装复杂软件。

## 快速开始

### 直接使用

1. **打开** [GitHub Pages](https://hsiao05.github.io/ReciteCloze/)
2. 粘贴要背诵的文本
3. 选择「自定义 DS API」，填入自己的 DeepSeek API Key，点击「生成标注语料」或「隔字生成」。**所有操作都在本地运行，不会在网络上泄露 API Key**
4. 切换到「背诵」页 → 开始交互练习

### 本地使用

1. **下载** [`index.html`](index.html)
2. `index.html` 中

> const DEVELOPER_API_KEY = "";

  填入自己的 DeepSeek API Key。不建议在网上公开包含自己 Key 的文件，以免造成一些不良后果。

3. **双击用浏览器打开**
4. 粘贴要背诵的文本 → 选择「开发者的 API」，点击「生成标注语料」或「隔字生成」
5. 切换到「背诵」页 → 开始交互练习

## 文本背诵怎么用

### 1. 进入配置页

打开工具后，顶部有三个页面：

- 卡片
- 配置
- 背诵

要做新的背诵材料，先点击 `配置`。

### 2. 输入源语料

在 `文本背诵` 区域里找到 `源语料`，把要背的内容粘贴进去。

可以粘贴普通文本，也可以粘贴简单的 Markdown。工具会尽量清理一些 Markdown 标记，比如标题、引用、加粗、链接等。

### 3. 选择生成方式

有两种方式：

- `生成标注语料`：调用 DeepSeek API，让模型判断重点。
- `隔字生成`：不用 API，用本地规则生成遮盖。

如果你只是想快速试试，建议先点 `隔字生成`。

如果你想让遮盖更像考试重点，可以使用 DeepSeek API。

### 4. 使用 DeepSeek API

在 API 模式里可以看到：

- `开发者的 API`
- `自定义 DS API`

如果你是普通使用者，建议选择 `自定义 DS API`，然后填写自己的 DeepSeek API Key、模型和 API 端点。

默认 API 端点是：

```text
https://api.deepseek.com/chat/completions
```

可选模型里有：

```text
deepseek-v4-flash
deepseek-v4-pro
```

填写好以后点击 `生成标注语料`。生成成功后会自动进入背诵页。

重要提醒：如果你要把项目开源，不要把自己的 API Key 写进公开代码里。更安全的做法是让使用者自己在页面里填写。

### 5. 修改标注结果

生成后，右侧会出现 `标注结果`。这个区域是可以编辑的。

标注规则大概是这样：

- `⟦...⟧` 表示结构保护，一般不遮盖。
- `⟬...⟭` 表示一层核心，优先背。
- `⟪...⟫` 表示二层扩展，背完核心后再背。
- `|` 表示最小语言单位，用来帮助工具拆分词块。

如果你觉得某些地方遮得不好，可以直接改标注。改完后点击 `解析并进入背诵`。

### 6. 开始背诵

进入 `背诵` 页后，底部有几个模式：

- `原文`：显示完整内容。
- `一层`：遮盖一层核心内容。
- `二层`：遮盖一层和二层内容。
- `重点`：只练手动标记的重点和高概率重点。
- `全遮`：尽量遮盖所有可背内容。

背诵时可以点击或拖动词块，让内容显示或隐藏。长按词块可以标记重点，再次长按或划过已标重点可以取消。

## 闪卡背诵怎么用

闪卡背诵适合单词表、问答表、概念表。

### 方法一：从 CSV 导入

CSV 文件第一行会被当作字段名。比如：

```csv
单词,读音,释义
apple,/ˈæpəl/,苹果
book,/bʊk/,书
```

使用步骤：

1. 进入 `配置`。
2. 找到 `闪卡背诵`。
3. 在 `CSV 导入闪卡` 里点击 `选择 CSV`。
4. 选择你的 CSV 文件。
5. 工具会读取字段和行数。
6. 选择起始行和结束行。
7. 勾选哪些字段显示在正面，哪些字段显示在反面。
8. 点击 `生成 Anki 卡片`。

建议 CSV 使用 UTF-8 编码。如果字段内容里有逗号，请用标准双引号包起来。

### 方法二：手动录入闪卡

如果没有 CSV，也可以手动做卡片。

使用步骤：

1. 进入 `配置`。
2. 找到 `手动闪卡`。
3. 在 `字段名` 里输入字段，例如：

```text
正面,反面
```

或者：

```text
单词,读音,释义
```

4. 点击 `应用字段`。
5. 点击 `添加卡片`，逐条填写内容。
6. 填完后点击 `生成闪卡`。

生成后会自动进入背诵页。

## 闪卡学习模式说明

闪卡进入背诵页后，底部的模式会变成：

- `正面`
- `反面`
- `重点`
- `总览`

### 正面

只显示你选中的正面字段。想看答案时点击 `显示反面`。

### 反面

显示正面和反面字段，然后根据记忆情况选择：

- `重来`：完全没记住，会很快再次出现。
- `困难`：有点记住，但还不熟。
- `良好`：基本记住。
- `简单`：很熟，会隔更久再复习。

点错了可以点 `回退`。

### 重点

这里只显示被标记过重点的记录。长按字段内容可以加入重点。

### 总览

总览会列出这组闪卡的所有记录，可以查看哪些是未学、巩固中、已学，也可以批量处理：

- 设为已学
- 设为未学
- 标重点
- 取消重点
- 清除学习数据
- 导出
- 导入学习数据

## 卡片管理

打开工具后的第一个页面是 `卡片` 页面。这里会显示浏览器中缓存过的背诵卡片。

可以做这些事情：

- 点击某张卡片继续背。
- 新建文件夹。
- 把卡片移动到文件夹。
- 多选管理卡片。
- 删除不需要的卡片。
- 导出卡片文件。
- 导入本地卡片文件。
- 从公开 GitHub 仓库、文件或 Gist 导入卡片。

导出的卡片是 JSON 文件。这个文件可以备份，也可以发给别人导入。

## 从 GitHub 导入卡片

卡片页面有一个 `从 GitHub 导入` 输入框。它支持这些公开链接：

- GitHub 仓库链接
- GitHub 文件链接
- GitHub 文件夹链接
- raw.githubusercontent.com 文件链接
- Gist 链接

要求：

- 链接必须是公开可访问的。
- 文件内容必须是本工具导出的 JSON 卡片文件。
- 如果是仓库链接，工具会查找仓库中的 JSON 文件。

如果导入失败，一般是下面几个原因：

- 仓库不是公开的。
- JSON 文件不是本工具导出的格式。
- GitHub API 暂时访问失败。
- 浏览器网络无法访问 GitHub。

## 数据保存在哪里

这个工具会把数据保存在浏览器的 `localStorage` 里。

主要保存这些内容：

- 页面设置
- API 设置
- 卡片列表
- 文件夹列表
- 闪卡学习状态
- 重点标记

这有一个好处：不需要服务器，也不需要账号。

也有几个限制：

- 换浏览器后数据不会自动同步。
- 换电脑后数据不会自动同步。
- 清理浏览器缓存可能会删除数据。
- 隐私模式下的数据可能关闭窗口后就没了。

所以重要卡片建议定期导出 JSON 文件备份。

## 导入和导出建议

### 导出全部或部分卡片

在 `卡片` 页点击 `导出文件` 后，工具会让你选择要导出的卡片。选择后点击上方的 `导出`。

每张卡片会导出成一个 JSON 文件。

### 导入卡片

点击 `导入文件`，选择之前导出的 JSON 文件即可。

工具会自动跳过重复卡片，避免同一张卡片反复导入。

### 导出闪卡内容和学习记录

在闪卡的 `总览` 页面点击 `导出`，会让你选择：

- `1`：只导出内容列表。
- `2`：只导出学习记录。
- `3`：内容和学习记录都导出。

如果你想把同一套闪卡发给别人，可以导出内容列表。如果你想在另一台设备继续自己的学习进度，可以同时导出学习记录。

## 常见问题

### 为什么我点生成标注语料没有反应

先检查有没有输入源语料。然后检查 API 模式。如果使用 `自定义 DS API`，需要填写 API Key。如果使用 `开发者的 API`，需要开发者在程序里提前配置好 Key。

### 为什么刷新页面后卡片还在

因为卡片保存在浏览器本地存储里。只要没有清理网站数据，一般刷新后还会在。

### 为什么换电脑后卡片没了

因为数据没有上传到服务器。换电脑前需要先导出 JSON 文件，再在新电脑上导入。

### 为什么 CSV 导入乱码

通常是 CSV 编码问题。建议把 CSV 保存为 UTF-8。Excel 导出的 CSV 有时不是 UTF-8，可以用编辑器（如 VSCode、Windows 记事本）重新保存。

### 这个工具会上传我的背诵内容吗

普通本地操作不会上传。使用 DeepSeek 生成标注时，源语料会发送给你配置的 API 端点。使用 GitHub 导入时，工具会访问你填写的 GitHub 链接。

## 开发说明

这个项目目前是纯前端实现，主要依赖浏览器能力：

- HTML 页面结构
- CSS 样式
- JavaScript 交互逻辑
- localStorage 本地保存
- fetch 请求 API 和 GitHub 文件
- File API 导入本地文件
- Blob 下载 JSON 文件

因为没有构建流程，所以也比较容易修改。改完 `index.html` 后刷新浏览器就能看效果。
